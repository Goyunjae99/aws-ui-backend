[목표]
현재 FastAPI 백엔드(main.py)를 "하드코딩 기반 Mock Runner" 방식으로 확장해,
UI에서 Start Provisioning을 누르면 실제 배포 대신 Mock 작업이 비동기로 진행되고,
진행 상태/로그/결과 리소스가 DB에 저장되며 history/monitoring에서 확인 가능하게 만들고 싶다.

[절대 지켜야 할 제약(매우 중요)]
1) 기존 라우트 경로/HTTP 메서드/응답 JSON의 큰 틀은 유지한다.
2) 기존 DB 모델을 가능한 한 수정하지 말 것. (컬럼 추가 수준만)
3) 기존 템플릿(templates/*)과 프론트 코드는 건드리지 말 것.
4) 기능 단순화/삭제 금지. 기존 동작은 유지하면서 확장만 한다.
5) 모든 변경 사항은 실행 가능한 상태로 커밋 수준의 코드로 반영한다.
6) 코드 수정이 있으면 "수정된 파일 전체 코드"를 제공한다.

[구현 요구사항]
A) Provisioning 작업을 비동기로 처리 (POST /api/provision → 즉시 응답, 백그라운드 Mock)
B) 하드코딩 설정을 config.py CONFIG로 한 곳에 모으기
C) GET /api/history, GET /api/monitoring/my-resources 에서 상태/리소스 확인 가능
D) details 스키마: input, status, logs, resources, error
E) services/provisioner.py, services/runners/mock_runner.py, config.py 분리
